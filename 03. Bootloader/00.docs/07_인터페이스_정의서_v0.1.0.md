# 인터페이스 정의서

- 프로젝트: NU-HC UART-BLE 부트로더
- 문서 버전: v0.1.0
- 기준일: 2026-06-18
- 작성자: 이한서

## 1. 문서 목적

본 문서는 부트로더 시스템을 구성하는 장치 간의 물리·논리 인터페이스를 정의한다. 하드웨어 핀 매핑, UART 레지스터 설정, 신호 코드, NU-HC AT 명령 인터페이스, 호스트 프로그래밍 인터페이스를 포함한다.

## 2. 인터페이스 개요

```text
[Linux Host] --(USB)-- [CP210x] --(UART)-- [NU-HC #1]
                                                │ BLE
                                            [NU-HC #2] --(UART1)-- [ATmega1281]
                                                                         │
                                                                    [LED: PORTC]
```

| 구간 | 인터페이스 | 물리 매체 |
| --- | --- | --- |
| Host ↔ NU-HC #1 | UART (19200/8N1) | USB(CP210x) |
| NU-HC #1 ↔ NU-HC #2 | BLE (NUS Throughput) | 무선 |
| NU-HC #2 ↔ ATmega1281 | UART1 (19200/8N1) | 직결 배선 |
| ATmega1281 ↔ 디버거 | UART0 (디버그) / JTAG | 직결 배선 |

## 3. ATmega1281 핀 인터페이스

| 핀 | 포트 | 기능 | 방향 | 연결 대상 |
| --- | --- | --- | --- | --- |
| RXD1 | PD2 | UART1 수신 | 입력 | NU-HC #2 TX |
| TXD1 | PD3 | UART1 송신 | 출력 | NU-HC #2 RX |
| RXD0 | PE0 | UART0 수신 | 입력 | 디버그 호스트 TX |
| TXD0 | PE1 | UART0 송신 | 출력 | 디버그 호스트 RX |
| PC0~PC7 | PORTC | 상태 LED | 출력 | LED 표시 |
| GND | — | 공통 접지 | — | NU-HC #2 GND |

UART1은 NU-HC(BLE 데이터 경로), UART0은 디버그 출력 전용으로 분리한다. NU-HC #2 TX는 반드시 MCU RXD1(PD2)에 교차 연결하고 GND를 공통으로 둔다.

## 4. UART 파라미터

| 항목 | 값 | 비고 |
| --- | --- | --- |
| Baudrate | 19200 bps | `UBRR=51` @ 16 MHz, 오차 0.2% |
| Data bits | 8 | `UCSZn1:0 = 11` |
| Parity | None | — |
| Stop bits | 1 | — |
| Flow control | 없음 | XON/XOFF 미사용 |

### 4.1 ATmega1281 레지스터 설정

| 레지스터 | 값 | 의미 |
| --- | --- | --- |
| `UBRRnH` | `0` | 보레이트 상위 |
| `UBRRnL` | `51` | 보레이트 하위 (19200 @ 16 MHz) |
| `UCSRnB` | `RXENn \| TXENn \| RXCIEn` | 송수신 + RX 완료 인터럽트 |
| `UCSRnC` | `UCSZn1 \| UCSZn0` | 8비트, 무패리티 |
| `UDRIEn` | 동적 | 송신 데이터 있을 때만 set, 큐 비면 clear |

115200 bps는 16 MHz에서 `U2X=1`이어도 약 +2.1% 오차로 262 byte 스트림이 깨져 사용하지 않는다.

## 5. 호스트(Linux) 시리얼 인터페이스

| 항목 | 값 |
| --- | --- |
| 장치 | `/dev/ttyUSB0` (CP210x) |
| 권한 | `dialout` 그룹 |
| 모드 | raw (`cfmakeraw`) |
| 속도 | `B19200` (입력/출력) |
| `VTIME` | 20 (2초) |
| `VMIN` | 0 |
| 비동기 수신 | `O_ASYNC` + SIGIO 핸들러 |

호스트는 응답(ACK/NAK/'R')을 SIGIO 비동기 신호로 수신하며, 송신 중에는 `F_SETFL, 0`으로 SIGIO를 잠시 비활성화한다.

## 6. 신호·메시지 인터페이스

| 신호 | 값 | 방향 | 의미 |
| --- | --- | --- | --- |
| `'S'` | `0x53` | Host → MCU | 핸드셰이크 시작 |
| `'R'` | `0x52` | MCU → Host | 핸드셰이크 준비 완료 |
| START_CODE | `0xEF` | Host → MCU | 패킷 시작 |
| MSG_DATA | `0x01` | Host → MCU | 데이터 페이지 |
| MSG_END | `0x02` | Host → MCU | 전송 종료 |
| ACK | `0x06` | MCU → Host | 정상 |
| NAK | `0x15` | MCU → Host | 오류/재전송 |

패킷 구조의 상세는 「프로토콜 명세서」를 따른다.

## 7. NU-HC AT 명령 인터페이스

NU-HC 모듈은 Command Mode에서 AT 명령으로 제어하고, Throughput Mode에서 데이터를 투명 전송한다.

| 동작 | 명령/방법 | 비고 |
| --- | --- | --- |
| 응답 확인 | `AT` → `OK` | 연결 확인 |
| BLE 연결 | `AT+LECCONN=<MAC><type>` | 상대 주소 지정 |
| 연결 상태 | `+GATTSTAT=0,3` | 연결 완료 표시 |
| Throughput 진입 | `AT+TPMODE=1` | 투명 전송, 기본값 1 |
| Command 복귀 | `+++` | 앞뒤 1초 guard time |

| 항목 | 값 |
| --- | --- |
| BLE 전송 방식 | write-without-response |
| flush 조건 | 182 byte 누적 또는 16 ms 경과 |
| 미연결 시 동작 | raw 입력을 AT로 해석, `ERROR\r\n` 응답 |

VS Code Serial Monitor(Line Ending CRLF)로 AT 명령을 직접 입력할 수 있다.

## 8. 메모리 인터페이스 (Flash)

| 영역 | 주소 범위 | 용도 |
| --- | --- | --- |
| Application | `0x00000` ~ `0x1DFFF` | 갱신 대상 응용 펌웨어 |
| Bootloader | `0x1E000` ~ | 부트로더 (보호) |

| 항목 | 값 |
| --- | --- |
| 페이지 크기 | 256 byte |
| 기록 방식 | SPM (`boot_page_erase/fill/write`) |
| 보호 정책 | `write_addr >= 0x1E000` 기록 거부 |
| 부트로더 코드 크기 | 약 1638 byte (.text) |
| 부트로더 RAM 사용 | 약 1032 byte (.bss, 링버퍼 4×256 포함) |

## 9. 프로그래밍 인터페이스 (개발 단계)

| 인터페이스 | 도구 | 용도 |
| --- | --- | --- |
| JTAG | J-Link (Microchip Studio) | 부트로더 최초 플래시·디버그 |
| UART0 | 시리얼 터미널 | 디버그 출력 확인 |
| UART1 | BLE 경로 | 펌웨어 무선 갱신(운영) |

부트로더 적재 후 응용 펌웨어 갱신은 JTAG가 아닌 본 프로토콜(UART1/BLE)로 수행한다.

## 10. 인터페이스 제약

| ID | 제약 |
| --- | --- |
| IF-C-001 | NU-HC #2 TX ↔ MCU RXD1(PD2) 교차 연결 및 GND 공통 필수 |
| IF-C-002 | 모든 UART 구간 19200/8N1로 통일 |
| IF-C-003 | DTR/RTS 물리 리셋은 BLE를 통과하지 못하므로 인밴드 핸드셰이크 사용 |
| IF-C-004 | 데이터 전송은 NU-HC Throughput Mode + BLE 연결 상태에서만 가능 |
| IF-C-005 | 시리얼 포트는 한 번에 한 프로세스만 점유 (터미널·송신 도구 동시 사용 금지) |
| IF-C-006 | 청크 크기는 NU-HC flush(182 byte/16 ms)에 맞춰 조정 |
