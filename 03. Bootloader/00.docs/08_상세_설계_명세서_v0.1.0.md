# 상세 설계 명세서

- 프로젝트: NU-HC UART-BLE 부트로더
- 문서 버전: v0.1.0
- 기준일: 2026-06-18
- 작성자: 이한서

## 1. 문서 목적

본 문서는 부트로더 시스템의 핵심 함수를 실제 구현 기준으로 상세히 명세한다. 호스트(`bl_sender`)와 MCU 양측의 함수별 입력·출력·동작·주의사항을 다룬다.

## 2. MCU 함수 명세

### 2.1 main / run_bootloader

| 항목 | 내용 |
| --- | --- |
| 초기화 | `DDRC=0xFF`, `PORTC=0`, `UART0_init`, `UART1_init` |
| 벡터 재배치 | `MCUCR=(1<<IVCE); MCUCR=(1<<IVSEL);` → 인터럽트 벡터를 부트 영역으로 |
| 인터럽트 허용 | `sei()` |
| 부트 대기 | `timeout=10000`, 루프마다 `_delay_ms(10)` → 약 100초 |
| 핸드셰이크 | `'S'` 수신 시 `USART1_RxFlush()` → `'R'` 송신 → `USART1_FlushTx()` → `run_bootloader()` |

`run_bootloader`는 `recv_packet` 결과에 따라 분기한다.

```text
result = recv_packet(&pkt); _delay_ms(100)
  MSG_DATA: write_addr >= 0x1E000 ? NAK+중단 : flash_page_write + ACK + (write_addr += 256) + LED 토글
  MSG_END : ACK + jump_to_app()
  기타    : NAK
```

`_delay_ms(100)`은 수신 직후 응답 송신 안정화를 위한 지연이며, 향후 `USART1_FlushTx` 기반으로 대체 가능하다.

### 2.2 recv_packet

| 항목 | 내용 |
| --- | --- |
| 입력 | `bl_data_packet_t *pkt` |
| 반환 | `MSG_DATA`/`MSG_END` (정상), `NAK` (CRC 불일치) |
| Step 0 | START_CODE 재동기: `do { raw[0]=USART1_Receive(); } while(raw[0]!=0xEF)` |
| Step 1 | 나머지 261 byte 수신 (`raw[1..261]`) |
| Step 2 | 파싱: `data_length = raw[2]|(raw[3]<<8)`, `data = raw[4..259]`, `crc16 = raw[260]|(raw[261]<<8)` |
| Step 3 | CRC 검증: `crc_buf = msg_type + raw[2] + raw[3] + data(256)`, `crc16(crc_buf, 259)` 비교 |
| 출력 | 일치 시 `msg_type` 반환, 불일치 시 `NAK` |

`USART1_Receive`는 큐가 빌 때까지 블로킹하므로, 마지막 byte가 늦게 도착해도 262 byte를 모두 모은다.

### 2.3 crc16

| 항목 | 내용 |
| --- | --- |
| 알고리즘 | CRC-16/CCITT-FALSE |
| 초기값 | `0xFFFF` |
| 다항식 | `0x1021` |
| 동작 | byte마다 상위 정렬 XOR 후 8회 시프트, MSB가 1이면 다항식 XOR |

```c
crc ^= ((uint16_t)data[i] << 8);
for (j=0; j<8; j++)
    crc = (crc & 0x8000) ? (crc << 1) ^ 0x1021 : (crc << 1);
```

호스트와 MCU가 동일 구현을 사용하므로 계산값이 일치한다.

### 2.4 flash_page_write

| 항목 | 내용 |
| --- | --- |
| 입력 | `uint32_t addr`, `uint8_t *buf` (256 byte) |
| 1단계 Erase | `boot_page_erase(addr)` → `SPMEN` 클리어 대기 |
| 2단계 Fill | 256 byte를 word(2 byte) 단위로 `boot_page_fill(i, word)` |
| 3단계 Write | `boot_page_write(addr)` → `SPMEN` 클리어 대기 |
| 4단계 RWW | `boot_rww_enable()`로 응용 영역 재활성화 |
| 인터럽트 | 진입 시 `cli()`, 종료 시 `sei()` |

word는 little-endian으로 조합한다: `word = buf[i] | (buf[i+1] << 8)`.

### 2.5 jump_to_app

| 항목 | 내용 |
| --- | --- |
| 인터럽트 차단 | `cli()` |
| 벡터 복원 | `MCUCR=(1<<IVCE); MCUCR=(0<<IVSEL);` → 벡터를 응용 영역으로 |
| UART 비활성화 | `UCSR0B = 0x00` |
| 점프 | `asm volatile("jmp 0x0")` |

부트 진입 시 set한 `IVSEL`을 다시 0으로 되돌려, 응용 프로그램이 자신의 인터럽트 벡터를 사용하도록 한다.

## 3. UART 드라이버 명세

### 3.1 인터럽트 수신 (RX)

| 항목 | 내용 |
| --- | --- |
| ISR | `USARTn_RX_vect` |
| 동작 | `UDRn` 읽어 원형큐(`rxn_buf`) 적재, 가득 차면 폐기 |
| 큐 크기 | 256 byte, `BUF_MASK = 255` |
| 인덱스 | `uint8_t` (AVR 원자적 접근) |

### 3.2 인터럽트 송신 (TX)

| 항목 | 내용 |
| --- | --- |
| ISR | `USARTn_UDRE_vect` |
| 동작 | 큐 비면 `UDRIEn` 클리어, 아니면 큐에서 `UDRn` 송신 |
| 송신 함수 | `USARTn_Transmit`: 큐에 적재 후 `UDRIEn` set |
| 핵심 | TX 인터럽트는 송신 데이터 있을 때만 활성, 큐 비면 자동 비활성 |

### 3.3 보조 함수

| 함수 | 동작 |
| --- | --- |
| `USARTn_Receive` | 큐 빌 때까지 블로킹 후 1 byte 반환 |
| `USARTn_Available` | 큐에 데이터 있으면 비0 반환 |
| `USART1_RxFlush` | `cli()` 보호 하에 `rx1_tail = rx1_head` (큐 비우기) |
| `USART1_FlushTx` | TX 큐 빌 때까지 + `TXC1` 완료까지 대기 |

`USART1_RxFlush`는 핸드셰이크 직후 1회만 호출한다. 매 패킷마다 호출하면 연속 전송 시 쌓인 다음 패킷을 폐기할 수 있다.

## 4. 호스트 함수 명세

### 4.1 uart_open

| 항목 | 내용 |
| --- | --- |
| 입력 | 포트 경로 (`/dev/ttyUSB0`) |
| 설정 | `cfmakeraw`, `B19200`, `VTIME=20`, `VMIN=0` |
| 비동기 | `SIGIO` 핸들러 등록, `F_SETOWN`, `O_ASYNC` |
| 반환 | 파일 디스크립터 |

### 4.2 sigio_handler

| 입력 byte | 동작 |
| --- | --- |
| `'R'` (0x52) | `ready_recv = 1` |
| `ACK` (0x06) | `feedback = 1` |
| `NAK` (0x15) | `feedback = 0` |
| 기타 | 로그만 출력 |

### 4.3 wait_ready

| 항목 | 내용 |
| --- | --- |
| 동작 | `'S'`를 200 ms 주기로 송신하며 `ready_recv` 대기 |
| 타임아웃 | 30회 (약 6초) |
| 성공 | `ready_recv` set 시 1 반환 |

현재 코드에는 DTR_Reset 후속 처리가 남아 있으나, `return` 도달로 실행되지 않는 죽은 코드이다. 정리 대상이다.

### 4.4 make_packet

| 항목 | 내용 |
| --- | --- |
| 입력 | `msg_type`, `data`, `data_len` |
| 동작 | `data` 256 byte를 `0xFF`로 초기화 후 복사, CRC16 계산 |
| CRC 범위 | `msg_type + (data_len LE 2byte) + data(256)` |

### 4.5 send_packet

| 항목 | 내용 |
| --- | --- |
| 청크 송신 | 20 byte 단위 `write` + `tcdrain` + `usleep(20000)` |
| SIGIO 제어 | 송신 중 `F_SETFL,0`, 송신 후 `O_ASYNC` |
| ACK 대기 | 50000회 × `usleep(100)` ≈ 5초 |
| 재전송 | NAK/타임아웃 시 반복, `MAX_RETRY=3` |
| 반환 | ACK 1, 실패 0 |

### 4.6 upload

| 항목 | 내용 |
| --- | --- |
| 동작 | 페이지마다 256 byte 읽어 `make_packet(MSG_DATA)` → `send_packet` |
| 종료 | 모든 페이지 후 `make_packet(MSG_END)` → `send_packet` |
| 실패 처리 | `send_packet` 실패 시 중단·반환 |

## 5. 자료구조

```c
typedef struct {
    uint8_t  start_code;        /* 0xEF */
    uint8_t  msg_type;          /* 0x01=DATA, 0x02=END */
    uint16_t data_length;       /* 256, little-endian */
    uint8_t  data[256];         /* 페이지 데이터 */
    uint16_t crc16;             /* CRC16, little-endian */
} __attribute__((packed)) bl_data_packet_t;   /* sizeof = 262 */
```

| 전역 변수 (호스트) | 의미 |
| --- | --- |
| `uart_fd` | 시리얼 파일 디스크립터 |
| `feedback` | -1=미수신, 0=NAK, 1=ACK |
| `ready_recv` | 핸드셰이크 'R' 수신 플래그 |

## 6. 리소스 사용량 (실측)

| 항목 | 값 |
| --- | --- |
| 부트로더 코드 (.text) | 약 1638 byte |
| RAM 사용 (.bss) | 약 1032 byte |
| RAM 내역 | RX/TX 원형큐 4 × 256 byte = 1024 byte + 인덱스·변수 |
| 부트로더 위치 | 0x1E000 |
| 시험 바이너리 | `LED_blink.bin` 1050 byte = 5 페이지 |
