# 기능 명세서

- 프로젝트: NU-HC UART-BLE 부트로더
- 문서 버전: v0.1.0
- 기준일: 2026-06-18
- 작성자: 이한서

## 1. 개요

본 문서는 부트로더 시스템의 기능을 호스트(`bl_sender`)와 대상 MCU(ATmega1281) 양측으로 나누어 상세히 정의한다. 기능은 전송 프로토콜, 핸드셰이크, 수신·검증, 기록, 응답, 점프의 단계로 구성된다.

## 2. 프로토콜 정의

### 2.1 패킷 구조 (262바이트 고정)

| 필드 | 크기 | 값/설명 |
| --- | --- | --- |
| START_CODE | 1 | 0xEF |
| MSG_TYPE | 1 | 0x01=DATA, 0x02=END |
| DATA_LENGTH | 2 | 256, little-endian |
| DATA | 256 | 펌웨어 페이지 데이터(부족분 0xFF 패딩) |
| CRC16 | 2 | CCITT-FALSE, little-endian |

### 2.2 메시지/응답 코드

| 코드 | 값 | 의미 |
| --- | --- | --- |
| MSG_DATA | 0x01 | 데이터 페이지 패킷 |
| MSG_END | 0x02 | 전송 종료, 점프 요청 |
| ACK | 0x06 | 정상 수신·기록 |
| NAK | 0x15 | 오류, 재전송 요구 |

### 2.3 CRC16 사양

| 항목 | 값 |
| --- | --- |
| 다항식(polynomial) | 0x1021 |
| 초기값(init) | 0xFFFF |
| 계산 범위 | MSG_TYPE + DATA_LENGTH + DATA(256) = 259바이트 |
| 바이트 순서 | little-endian 전송 (예: 0xF404 → `04 F4`) |

## 3. 호스트(bl_sender) 기능

| ID | 기능 | 설명 |
| --- | --- | --- |
| HF-001 | 바이너리 로드 | bin 파일을 열고 크기로 페이지 수 산출 |
| HF-002 | 포트 오픈 | `uart_open`으로 19200/8N1 raw 모드 설정, SIGIO 비동기 수신 등록 |
| HF-003 | 핸드셰이크 | `'S'`를 주기적으로 송신하고 MCU의 `'R'` 응답을 대기 |
| HF-004 | 패킷 생성 | `make_packet`으로 262바이트 패킷 구성 및 CRC 계산 |
| HF-005 | 청크 전송 | `send_packet`이 20바이트 단위로 분할 송신 (20ms 간격) |
| HF-006 | 응답 처리 | SIGIO 핸들러가 ACK/NAK/READY를 분류하여 플래그 갱신 |
| HF-007 | 재전송 | NAK 또는 타임아웃 시 최대 3회 재전송 |
| HF-008 | 종료 처리 | 모든 페이지 후 MSG_END 송신, 점프 확인 |

### 3.1 호스트 송신 흐름

```text
bin 열기 → 페이지 수 산출 → 포트 오픈 → wait_ready('S'→'R')
  → for each page: make_packet → send_packet(청크) → ACK 대기
       ├ ACK  → 다음 페이지
       └ NAK/timeout → 재전송(최대 3회)
  → MSG_END 송신 → 완료
```

## 4. MCU 부트로더 기능

| ID | 기능 | 설명 |
| --- | --- | --- |
| MF-001 | 초기화 | UART0/UART1 init, 인터럽트 벡터 재배치(IVSEL), 전역 인터럽트 허용 |
| MF-002 | 부트 대기 | 약 100초 동안 `'S'` 시작 신호 대기 |
| MF-003 | 핸드셰이크 | `'S'` 수신 시 RX 큐 비우고 `'R'` 응답, 전송 완료 대기 |
| MF-004 | 패킷 수신 | START_CODE 재동기 후 262바이트 수신 |
| MF-005 | 무결성 검증 | CRC16 재계산 후 수신값과 비교 |
| MF-006 | 페이지 기록 | `flash_page_write`로 응용 영역에 기록 |
| MF-007 | 응답 송신 | 정상 ACK, 오류 NAK를 UART1로 전송 |
| MF-008 | 영역 보호 | 기록 주소가 0x1E000 이상이면 거부 |
| MF-009 | 응용 점프 | MSG_END 시 `jump_to_app` 실행 |

### 4.1 수신·재동기 동작 (MF-004)

```text
recv_packet:
  do { raw[0] = USART1_Receive(); } while(raw[0] != 0xEF)   // START_CODE 재동기
  for i in 1..261: raw[i] = USART1_Receive()                // 나머지 261바이트
  파싱(MSG_TYPE, DATA_LENGTH, DATA, CRC16)
  CRC16 재계산 == 수신 CRC ?  → MSG_TYPE 반환 : 오류 반환
```

START_CODE 재동기는 핸드셰이크 잔재나 노이즈가 선행해도 0xEF를 찾아 정렬을 복구하는 안전망이다.

### 4.2 응답·기록 동작 (MF-005~MF-009)

```text
run_bootloader (반복):
  result = recv_packet()
  if result == MSG_DATA:
      if write_addr >= 0x1E000: NAK; 중단      // 부트 영역 보호
      flash_page_write(write_addr, data); ACK; write_addr += 256
  elif result == MSG_END:
      ACK; jump_to_app()
  else:
      NAK
```

## 5. UART 드라이버 기능

| ID | 기능 | 설명 |
| --- | --- | --- |
| UF-001 | 채널 분리 | UART1=NU-HC 데이터, UART0=디버그 출력 |
| UF-002 | 인터럽트 RX | `USARTn_RX_vect`가 256바이트 원형큐에 적재 |
| UF-003 | 인터럽트 TX | `USARTn_UDRE_vect`가 큐에서 송신, 빌 때 UDRIE off |
| UF-004 | 블로킹 수신 | `USARTn_Receive`가 큐가 빌 때까지 대기 후 반환 |
| UF-005 | 큐 비우기 | `USART1_RxFlush`가 인터럽트 보호 하에 RX 큐 초기화 |
| UF-006 | 송신 완료 대기 | `USART1_FlushTx`가 TX 큐와 TXC 완료까지 대기 |

## 6. 핸드셰이크 시퀀스

```text
Host(bl_sender)                         MCU(ATmega1281)
   │  'S' 반복 송신  ───────────────────▶ │  부트 대기 중 'S' 수신
   │                                      │  USART1_RxFlush() (잔재 제거)
   │ ◀───────────────────  'R' 응답       │  USART1_Transmit('R')
   │  'R' 수신 → 패킷 전송 시작            │  USART1_FlushTx() 후 run_bootloader
   │  262B 패킷  ─────────────────────────▶ │  START_CODE 재동기 → 수신 → 검증
   │ ◀──────────────────  ACK/NAK         │  기록 후 응답
```

## 7. 예외·오류 처리

| 상황 | 호스트 동작 | MCU 동작 |
| --- | --- | --- |
| CRC 불일치 | NAK 수신 → 재전송 | NAK 송신 |
| START_CODE 어긋남 | (영향 없음) | 0xEF까지 폐기 후 재정렬 |
| ACK 미수신 | 타임아웃 → 재전송(≤3) | — |
| 부트 영역 기록 시도 | 실패 보고 | NAK 후 중단 |
| 핸드셰이크 미수신 | 부트 타임아웃 종료 | 대기 종료 후 정지 |
