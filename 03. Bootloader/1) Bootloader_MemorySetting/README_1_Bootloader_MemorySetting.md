# 1) Bootloader_MemorySetting

부트로더 개발의 **첫 단계**로, AVR의 메모리 영역 개념을 확인하고 **부트 영역에서 응용 영역으로의 점프**가 동작하는지 검증한다. 부트로더의 본질인 "부트 영역에서 시작해 응용 영역으로 넘어간다"를 가장 단순한 형태로 구현한다.

> 환경: ATmega1281 (16 MHz) · Microchip Studio (AVR)

## 1. 개념: 부트 영역과 응용 영역

ATmega1281의 플래시는 두 영역으로 나뉜다.

```text
0x00000 ┌────────────────────────┐
        │  Application Section   │  ← 응용 펌웨어 (jmp 0x0의 목적지)
        │                        │
0x1E000 ├────────────────────────┤
        │  Boot Loader Section   │  ← 부트로더 (리셋 후 실행)
        │                        │
0x1FFFF └────────────────────────┘
```

BOOTRST 퓨즈를 켜면 리셋 후 **부트 영역**부터 실행된다. 부트로더는 자기 일을 마친 뒤 `jmp 0x0`으로 응용 영역에 제어를 넘긴다. 이 프로젝트는 그 전환을 LED로 확인한다.

---

## 2. 동작

```text
리셋 → 부트 영역 진입
  → PORTC LED 250ms 간격 10회 점멸 (부트 영역 실행 표시)
  → jmp 0x0 → 응용 영역으로 점프
```

LED가 10회 깜빡인 뒤 응용 영역으로 넘어가면, 부트→앱 전환이 정상 동작하는 것이다.

---

## 3. 코드 분석

```c
#include <avr/io.h>
#include <avr/interrupt.h>
#include <util/delay.h>

#define APP_START 0x0000          // 응용 영역 시작 주소

int main() {
    DDRC = 0xFF;                  // PORTC 전체 출력 설정

    for (int i = 0; i < 10; i++) {     // 부트 영역 실행 표시
        PORTC = 0xFF; _delay_ms(250);  // LED ON
        PORTC = 0x00; _delay_ms(250);  // LED OFF
    }

    asm volatile("jmp 0x0");      // 응용 영역(0x0000)으로 점프
}
```

| 구문 | 의미 |
| --- | --- |
| `DDRC = 0xFF` | PORTC 8핀을 모두 출력으로 — LED 제어 |
| `for` 루프 | 부트 영역 코드가 실행 중임을 시각적으로 표시 |
| `asm volatile("jmp 0x0")` | 인라인 어셈블리로 0x0000 절대 점프 |

> 이 단계에서는 인터럽트 벡터 재배치(IVSEL)나 스택 정리 없이 단순 점프만 한다. 정교한 점프(벡터 복원·UART 비활성화)는 프로젝트 5·7의 `jump_to_app`에서 다룬다.

## 4. BOOTSZ 퓨즈 설정

부트 영역의 크기와 위치는 **BOOTSZ 퓨즈**로 결정된다. Microchip Studio에서:

```text
Tools → Device Programming → Fuses
  BOOTSZ : 부트 영역 크기 선택 (예: 4096 words → 0x1E000~)
  BOOTRST: 켜면 리셋 후 부트 영역부터 실행
```

부트로더를 부트 영역에 플래시하려면, 빌드 출력의 `.text` 시작 주소가 부트 영역 주소(0x1E000)와 일치해야 한다. (Toolchain → Linker → `-Ttext`)

---

## 5. 확인 방법

| 단계 | 기대 결과 |
| --- | --- |
| 부트 영역에 플래시 | LED 250ms 간격 10회 점멸 |
| 응용 영역에 별도 코드 플래시 | 점멸 후 응용 코드 동작 |
| 점프 동작 | 10회 점멸 직후 응용 코드로 전환 |

응용 영역이 비어 있으면(0xFF) 점프 후 NOP 슬라이드로 다시 부트로 돌아올 수 있으니, 응용 영역에 식별 가능한 코드(다른 LED 패턴)를 넣어 확인하면 명확하다.
